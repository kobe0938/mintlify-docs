Sometimes a pass/fail unit test cannot capture whether an agent produced a genuinely good result. This tutorial shows you how to replace a deterministic verifier with an LLM judge that scores the agent's output on subjective criteria and writes a numeric reward to `reward.json`. The full source for this example lives in [`examples/tasks/llm-judge-example`](https://github.com/laude-institute/harbor/tree/main/examples/tasks/llm-judge-example).

<Info>
  **Recommended: Rewardkit** — For a declarative approach to LLM-as-a-judge, see [Rewardkit](/rewardkit/judge-criteria). The tutorial below shows how to do it manually with a custom Python script.
</Info>

## Task overview

In this example the agent is asked to write a funny poem. A verifier script sends the poem to Claude, which returns a `funny_score` between 0 and 1. That score is written to `reward.json` so Harbor can record it as a named metric.

<Steps>
  <Step title="Review the directory structure">
    The task follows the standard Harbor layout, with the judge script living inside `tests/` alongside `test.sh`.

    ```
    llm-judge-example/
    ├── instruction.md
    ├── task.toml
    ├── environment/
    │   └── Dockerfile
    ├── solution/
    │   └── solve.sh
    └── tests/
        ├── test.sh
        └── llm_judge.py
    ```
  </Step>
  <Step title="Pass API keys to the verifier via task.toml">
    The `[verifier.env]` section in `task.toml` injects environment variables into the container during verification — after the agent runs. Use `${VAR}` syntax to read secrets from the host environment, or supply literal values directly.

    ```toml title="task.toml"
    [verifier]
    timeout_sec = 900.0

    [verifier.env]
    ANTHROPIC_API_KEY = "${ANTHROPIC_API_KEY}"   # from host env
    MODEL_NAME = "claude-haiku-4-5"              # literal value
    ```

    This keeps API keys out of your task source while still making them available at verification time.
  </Step>
  <Step title="Write the judge script">
    The judge uses the Anthropic API with [structured outputs](https://docs.anthropic.com/en/docs/build-with-claude/structured-outputs) to return a validated, type-safe score.

    ```python title="tests/llm_judge.py"
    # /// script
    # dependencies = [
    #   "anthropic>=0.75.0",
    #   "pydantic==2.12.5",
    # ]
    # ///

    import json
    import os
    from pathlib import Path

    from anthropic import Anthropic, transform_schema
    from pydantic import BaseModel, Field


    class FunnyScoreResponse(BaseModel):
        funny_score: float = Field(..., ge=0.0, le=1.0)


    def main():
        poem = Path("/app/poem.txt").read_text()
        client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))
        schema = transform_schema(FunnyScoreResponse.model_json_schema())

        response = client.messages.create(
            model=os.getenv("MODEL_NAME"),
            max_tokens=1024,
            output_config={"format": {"type": "json_schema", "schema": schema}},
            messages=[
                {"role": "user", "content": f"Rate how funny this poem is from 0.0 to 1.0.\n\nPoem:\n{poem}"}
            ],
        )

        result = FunnyScoreResponse.model_validate_json(response.content[0].text)
        Path("/logs/verifier/reward.json").write_text(
            json.dumps({"funny": result.funny_score}, indent=2)
        )


    if __name__ == "__main__":
        main()
    ```

    The `test.sh` entry point simply invokes the judge with `uv run` to handle dependencies automatically:

    ```bash title="tests/test.sh"
    #!/bin/bash
    uv run /tests/llm_judge.py
    ```
  </Step>
  <Step title="Write reward.json">
    Instead of a binary `reward.txt`, this task writes a JSON object with named metrics. Harbor reads both formats — `reward.txt` first, then `reward.json` as a fallback.

    ```json
    { "funny": 0.75 }
    ```

    You can return multiple scores in a single JSON object:

    ```json
    { "creativity": 0.9, "humor": 0.7, "grammar": 1.0 }
    ```

    Each key becomes a separately tracked metric in Harbor's results.
  </Step>
  <Step title="Run the task">
    Export your API key, then run the task — first with the oracle to verify the setup, then with a real agent.

    ```bash
    # Prerequisites
    export ANTHROPIC_API_KEY="sk-ant-..."

    # Run with oracle (uses solution/solve.sh)
    harbor run -p examples/tasks/llm-judge-example -a oracle

    # Run with a real agent
    harbor run -p examples/tasks/llm-judge-example -a claude-code -m anthropic/claude-sonnet-4-5
    ```
  </Step>
</Steps>

## Adapting the pattern to your tasks

This pattern works with any LLM provider — swap the Anthropic client for OpenAI, Google, or any other SDK. Follow these four steps to adapt it:

1. Define a Pydantic model for the score or scores you need.
2. Write a prompt that instructs the LLM how to evaluate the agent's output.
3. Write the scores to `/logs/verifier/reward.json`.
4. Pass any required API keys via `[verifier.env]` in `task.toml`.

<Info>
  **Cost:** each verification call consumes API tokens. Factor this into your budgeting when running large jobs with an LLM judge.
</Info>
