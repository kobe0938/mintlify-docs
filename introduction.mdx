Harbor is a Python framework for evaluating and optimizing AI agents in containerized environments. It gives you a single, consistent interface for defining tasks, wiring up agents, and running evaluations — whether you are benchmarking a model on Terminal-Bench, generating supervised fine-tuning traces, or running reinforcement learning loops. Harbor handles the container lifecycle, parallel scheduling, and result collection so you can focus on the agents and tasks themselves.

## Why Harbor?

Running agent evaluations in containers is harder than it looks. You need to manage Docker images, coordinate parallel trials, collect rewards, and integrate with cloud sandbox providers when local compute runs out. Harbor was built specifically to solve these problems after the release of Terminal-Bench revealed that teams were spending more time on evaluation infrastructure than on agents.

Harbor gives you:

- **Simple, modular interfaces** for environments, agents, and tasks — swap any component without touching the rest of your setup.
- **Pre-integrated agents** so you can start evaluating immediately without writing boilerplate.
- **A dataset registry** with popular benchmarks ready to download and run in one command.
- **Cloud sandbox integrations** for horizontal scaling when local Docker is not enough.
- **Post-training tooling** to generate SFT traces and run RL optimization directly from evaluation rollouts.

## Supported agents

Harbor comes with pre-integrated support for the most widely used CLI agents so you can start benchmarking without any extra setup:

- **Terminus-2** — Harbor's own high-performance terminal agent
- **Claude Code** — Anthropic's agentic coding assistant
- **Copilot CLI** — GitHub Copilot's command-line agent
- **Gemini CLI** — Google's command-line agent
- **Codex CLI** — OpenAI's terminal coding agent
- **OpenHands** — the open-source generalist agent from All-Hands AI
- **Mini-SWE-Agent** — a lightweight SWE-Agent variant for software engineering tasks

You can also integrate any custom agent by implementing the `BaseAgent` or `BaseInstalledAgent` interface. See the [Agents](/agents/overview) documentation for details.

## Supported cloud providers

When local Docker is not fast enough, Harbor can distribute trials across any of the following cloud sandbox providers:

| Provider | Notes |
|---|---|
| Daytona | Recommended for high-concurrency runs |
| Modal | Serverless GPU and CPU containers |
| E2B | Lightweight sandboxes for coding agents |
| Runloop | Managed container sandboxes |
| Tensorlake | Native Harbor integration |
| LangSmith | LangChain's observability-enabled sandboxes |
| Blaxel | Edge-optimized container runtime |
| EC2 | Self-managed AWS instances |
| Beam | Serverless cloud compute |

<Note>
  Cloud sandboxes turn agent trials from compute-bound to I/O-bound workloads, which means you can safely run dozens or hundreds of trials concurrently. We strongly recommend using a cloud provider for any serious benchmarking effort.
</Note>
