Dataset metrics define how Harbor aggregates per-task rewards into a summary score at the end of a run. By default, Harbor averages all task rewards together and treats missing rewards as zero. You can override this behavior by adding a `metric.py` script to your dataset — giving you full control over how performance is reported.

## When to use a custom metric

The default average reward is sufficient for most evaluations. Consider writing a custom metric when you need to:

- Treat missing rewards differently than zero (for example, exclude them from the denominator)
- Compute multiple named metrics from the same reward file (for example, both `mean` and `pass_rate`)
- Apply task-specific weights or group tasks into sub-scores

## Add a metric script to your dataset

If you are working with a `dataset.toml`, run the following command to create a `metric.py` file and automatically add it to the `[[files]]` section of your `dataset.toml`:

```bash
harbor init --dataset "<org>/<name>" --with-metric
```

For local datasets, Harbor picks up `metric.py` automatically without any manifest changes.

## How Harbor invokes `metric.py`

Harbor calls your script with two flags after all tasks finish:

- `-i` / `--input-path` — path to a JSONL file where each line is a JSON reward object from one task
- `-o` / `--output-path` — path where your script must write a JSON object containing your computed metrics

Your script must read the rewards, compute whatever aggregates you want, and write the result to the output path.

## Example: mean reward with missing-reward handling

The following script mirrors Harbor's built-in behavior but treats `null` rewards as zero explicitly:

```python title="metric.py"
# /// script
# dependencies = []
# ///
# To add a dependency: uv add --script metric.py <dependency>

import argparse
import json
from pathlib import Path


def main(input_path: Path, output_path: Path):
    rewards = []

    for line in input_path.read_text().splitlines():
        reward = json.loads(line)
        if reward is None:
            rewards.append(0)
        elif len(reward) != 1:
            raise ValueError(
                f"Expected exactly one key in reward dictionary, got {len(reward)}"
            )
        else:
            rewards.extend(reward.values())

    mean = sum(rewards) / len(rewards) if rewards else 0

    output_path.write_text(json.dumps({"mean": mean}))


if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument(
        "-i",
        "--input-path",
        type=Path,
        required=True,
        help="Path to a jsonl file containing rewards, one json object per line.",
    )
    parser.add_argument(
        "-o",
        "--output-path",
        type=Path,
        required=True,
        help="Path to a json file where the metric will be written as a json object.",
    )
    args = parser.parse_args()
    main(args.input_path, args.output_path)
```

## Multiple metric values

Your script can return more than one metric in the same output JSON object:

```json
{"mean": 0.81, "pass_rate": 0.5}
```

Harbor surfaces every key-value pair in the run summary, so you can track pass rate alongside mean reward without writing a second script.

## Publishing a dataset with a custom metric

When you publish your dataset, Harbor automatically bundles `metric.py` alongside the tasks:

```bash
harbor publish "<path/to/dataset>" --public
```

Anyone who runs your published dataset will automatically use your custom metric — no extra setup required on their end.

<Tip>
  If you add or modify `metric.py` after a dataset is already published, re-run `harbor publish` to push the updated metric script. The new version will be tagged `latest` and picked up on subsequent runs.
</Tip>
